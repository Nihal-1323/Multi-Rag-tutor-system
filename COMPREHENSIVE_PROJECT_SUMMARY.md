## 📋 Executive Summary

This is a **production-ready Multi-Modal Retrieval-Augmented Generation (RAG) system** that processes text documents, images, and audio files to answer questions using semantic search and AI generation. The system has been transformed from a hardcoded, keyword-based implementation into a general-purpose, extensible platform.

**Architecture**: Clean Weaviate-only (no Pinecone) - fully local, GPU-optimized

---

## 🎯 Project Overview

### What This System Does

The Multi-Modal RAG system enables users to:
- **Upload** documents (PDF, TXT, MD, CSV), images (JPG, PNG, GIF, etc.), and audio files (MP3, WAV, etc.)
- **Ask questions** in natural language about the uploaded content
- **Receive intelligent answers** generated by combining semantic search with AI language models
- **Get source attribution** showing which documents were used to generate answers

### Key Innovation

**Before**: Hardcoded keyword matching with magic numbers
**After**: Semantic understanding with embedding-based search and intelligent fusion

The system now uses:
- **Semantic embeddings** instead of keyword matching
- **Cross-modal search** (text-to-image similarity using CLIP)
- **Intelligent ranking** based on modality compatibility
- **Adaptive fusion** that selects the best answer generation strategy
- **Explainable AI** with debug information at every step

## 🎨 Core Components

### 1. Query Understanding (`query_understanding.py`)

**Purpose**: Understand what the user is asking for

**Features**:
- Intent classification (visual_content, visual_attribute, text_retrieval, comparison, hybrid)
- Modality requirement detection (image_only, text_only, balanced, etc.)
- Entity extraction (filenames, concepts)
- Visual attribute detection (color, shape, size)
- Confidence scoring

**Example**:
```python
analysis = query_understanding.analyze("what color is the ball")
# QueryAnalysis(
#     intent=VISUAL_ATTRIBUTE,
#     modality_requirement=IMAGE_PRIMARY,
#     entities=["ball"],
#     visual_attributes=["color"],
#     confidence=0.9
# )
```

**Extensibility**: Can be replaced with ML models (BERT, fine-tuned classifiers)


### 2. Retrieval System (`retrieval.py`)

**Purpose**: Fetch relevant documents from different sources

**Retrievers**:

1. **SemanticRetriever**: Text documents
   - Uses sentence-transformers embeddings
   - Cosine similarity search
   - Fallback to keyword matching

2. **VisionRetriever**: Images
   - Uses CLIP embeddings
   - Cross-modal text-to-image similarity
   - Supports vision model descriptions

3. **AudioRetriever**: Audio files
   - Transcribes with Whisper
   - Embeds transcription as text
   - Semantic search on transcription

4. **GraphRetriever**: Knowledge graph
   - Traverses Neo4j graph
   - Finds related concepts
   - Boosts connected documents

5. **HybridRetriever**: Combines all
   - Weighted fusion based on query intent
   - Parallel retrieval from all sources
   - Normalized scoring

**Example**:
```python
hybrid = HybridRetriever([semantic, vision, audio, graph])
results = hybrid.retrieve(
    query="what color is the ball",
    modality_weights={"image": 1.0, "text": 0.4, "audio": 0.1},
    top_k=10
)
```

### 3. Reranking (`reranking.py`)

**Purpose**: Intelligently reorder results based on query understanding

**Features**:
- **Modality Compatibility Matrix**: Scores how well each result matches the query intent
- **Entity Matching Bonus**: Boosts results that mention query entities
- **Visual Attribute Bonus**: Boosts images for visual queries
- **Diversity Injection (MMR)**: Ensures variety in results
- **Explainable Reasoning**: Human-readable explanation for each score

**Compatibility Matrix**:
```python
{
    IMAGE_ONLY: {"image": 1.0, "text": 0.2, "audio": 0.1},
    TEXT_ONLY: {"image": 0.2, "text": 1.0, "audio": 0.3},
    BALANCED: {"image": 0.8, "text": 0.8, "audio": 0.8}
}
```

### 4. Fusion (`fusion.py`)

**Purpose**: Generate final answer by combining multi-modal sources

**Modes**:

1. **image_only**: Direct answer from vision model
   - Used when: High image confidence, visual query
   - Example: "what color is in the image" → "The image shows a red circle"

2. **text_only**: Answer from text documents
   - Used when: High text confidence, factual query
   - Example: "explain neural networks" → Answer from PDF

3. **hybrid**: Combine both modalities
   - Used when: Both have moderate confidence
   - Example: "compare the diagram to the text" → Combined answer

**Decision Logic**:
```python
if image_confidence > 0.7 and image_confidence > text_confidence * 1.5:
    mode = "image_only"
elif has_vision and image_conf > 0.3 and text_conf > 0.3:
    mode = "hybrid"
else:
    mode = "text_only"
```

### 5. Pipeline (`pipeline.py`)

**Purpose**: Orchestrate all components

**Flow**:
1. Query Understanding → Analyze intent
2. Adaptive Retrieval → Fetch with weighted modalities
3. Vision Processing → Generate image descriptions if needed
4. Intelligent Reranking → Score and reorder results
5. Adaptive Fusion → Generate final answer

**Usage**:
```python
pipeline = MultiModalRAGPipeline(
    document_store=document_store,
    knowledge_graph=knowledge_graph,
    ollama_url="http://localhost:11434",
    ollama_model="llama2",
    vision_service=vision_service
)

result = pipeline.process_query("what color is the ball", top_k=5)
```

## 🔄 System Transformation: Before vs After

### Query Processing Comparison

#### OLD SYSTEM (Hardcoded)
```python
# Brittle keyword matching
visual_keywords = ["color", "image", "picture"]
if any(kw in query.lower() for kw in visual_keywords):
    query_type = "VISION"
    score += 300  # Magic number!

# Monolithic code
if is_visual_query and is_image:
    score += 300
if is_visual_query and not is_image:
    score -= 150
```

**Problems**:
- Breaks for paraphrased queries
- Can't handle new domains
- No semantic understanding
- Magic numbers everywhere
- Tightly coupled code

#### NEW SYSTEM (General)
```python
# Semantic understanding
analysis = query_understanding.analyze(query)
# → Intent, modality, entities, confidence

# Embedding-based retrieval
query_embedding = embedding_service.embed_text(query)
results = retriever.retrieve_with_embeddings(query_embedding)

# Intelligent reranking
ranked = reranker.rerank(results, analysis)

# Adaptive fusion
answer = fusion_engine.fuse(query, ranked, analysis)
```

**Benefits**:
- Works for any domain
- Semantic understanding
- Learned embeddings
- Modular architecture
- Explainable results

## 🏗️ System Architecture Details

### Vector Storage Architecture

The system uses **Weaviate** for local, GPU-optimized vector storage:

```
┌─────────────────────────────────────────────────────────────┐
│                    DOCUMENT UPLOAD FLOW                      │
│                                                              │
│  User uploads document                                       │
│         ↓                                                    │
│  Extract text/Transcribe audio/Analyze image                │
│         ↓                                                    │
│  Generate embedding (384-dim or 512-dim)                    │
│         ↓                                                    │
│  ┌──────────────┬──────────────┐                           │
│  │   Memory     │   Weaviate   │                           │
│  │   (Cache)    │   (Local)    │                           │
│  │   ✅ Cached  │   ✅ Stored  │                           │
│  └──────────────┴──────────────┘                           │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                    QUERY PROCESSING FLOW                     │
│                                                              │
│  User query: "explain neural networks"                       │
│         ↓                                                    │
│  Generate query embedding                                    │
│         ↓                                                    │
│  Search Weaviate (local, fast)                              │
│         ↓                                                    │
│  Rank by similarity score                                    │
│         ↓                                                    │
│  Generate answer with LLM                                    │
└─────────────────────────────────────────────────────────────┘
```

### Core Components

#### 1. Weaviate Client (`backend/app/db/weaviate.py`)

**Purpose**: Local vector database for semantic search

**Features**:
- Docker-based deployment
- GPU-optimized similarity search
- Fast local queries (<200ms)
- Metadata filtering
- Schema management

#### 2. Semantic Retriever (`backend/app/core/retrieval.py`)

**Search Priority**:
1. **Weaviate** - Primary (embedding-based)
2. **In-Memory Cache** - Fallback
3. **Keyword Matching** - Last resort

**Code Flow**:
```python
def retrieve(self, query: str, top_k: int):
    # Try Weaviate first
    if weaviate_available:
        return search_weaviate(query)
    
    # Fallback to in-memory embeddings
    if embeddings_available:
        return search_in_memory(query)
    
    # Last resort: keyword matching
    return keyword_search(query)
```
